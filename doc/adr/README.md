# Architecture Decision Records

This directory holds the Architecture Decision Records (ADRs) for `lgtm`. Each ADR captures one significant, hard-to-reverse decision in Michael Nygard's format (Title, Status, Date, Context, Decision, Consequences). Records are numbered sequentially (`0001-...`, `0002-...`) and are immutable: once an ADR is merged it is never edited. When a decision changes, a new ADR is written with status `Accepted` (it may carry a "Supersedes ADR-XXXX" cross-link line, but that is a cross-link, not a status); the old ADR's Status is changed to `Superseded by ADR-YYYY`; and the two are cross-linked so the reasoning trail stays intact. ADRs are reviewed in the pull request that introduces the decision they document, per the Nautilus SDLC rule that irreversible choices (database choice, auth model, deploy target, runtime, language) are mandatory ADRs.
