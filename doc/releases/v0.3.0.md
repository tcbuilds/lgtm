# v0.3.0

## What changed

- Added native Codex hook support. `lgtm init --agent codex` installs the
  project hook wiring without changing Codex's private trust settings.
- Added Codex event adapters for prompt context, tool permission checks,
  post-tool checks, and stop decisions with exact JSON responses.
- Added optional `.lgtm/execpolicy.json` support for generating a Codex
  command backstop at `.codex/rules/lgtm.rules`.
- Added Codex end-to-end tests, idempotent hook initialization, and clear
  documentation of trust and platform limits.
- Fixed Codex trust-state reporting so `lgtm init --agent codex` explains when
  the project appears trusted and when the user should review `/hooks`.

## Affected users

Claude integrations keep their existing behavior. Codex users can opt in with
the new agent flag; existing Claude projects do not need migration.

## Upgrade

```sh
lgtm update
lgtm init --agent codex   # Codex projects only
```

The Codex setup is safe to re-run and preserves unrelated hook configuration.
