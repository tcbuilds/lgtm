## What changed

- Fixed Python gate discovery so `pytest.ini`, `tox.ini`, and declared `pytest` dependencies prove pytest without guessing unrelated tools.
- Ruff and Mypy commands now appear only when configured in `pyproject.toml` or declared in `requirements.txt`.
- Unknown Python tooling is omitted from `lgtm init` instead of forcing `--accept-guesses`; add explicit commands in `.lgtm/config.json` when needed.
- Added regression coverage for requirements-only and pytest-configured repositories.

## Verification

- `cargo test --locked --all-targets --all-features`: 316 passed.
- Real `~/owlet` dry-run: Python workspace reports only high-confidence `pytest`; frontend gates remain detected.

## Upgrade

```sh
lgtm update
```

Then rerun `lgtm init` in each repository that previously accepted guessed
commands. Review the generated config before committing it.
