# v0.3.1

## What changed

- Added Codex `PermissionRequest` support with exact allow/deny JSON and
  prohibited-command checks for configured approval policies.
- Added Codex `SubagentStart` and `SubagentStop` hooks, including standards
  context injection and protection against repeated stop continuations.
- Clarified Codex response semantics: Stop requests continuation, while
  PostToolUse feedback cannot undo a completed tool call.
- Added an execution-path capability matrix and documented incomplete
  `unified_exec` coverage instead of implying universal shell interception.
- Made generated Codex hooks prefer a stable absolute binary path when
  available, with restricted-`PATH` launch coverage.
- Added protocol drift tests, Codex end-to-end coverage, and an opt-in live
  conformance smoke test.

## Affected users

Claude behavior remains unchanged. Codex projects should rerun:

```sh
lgtm update
lgtm init --agent codex
```

Review new or changed hooks in Codex's `/hooks` screen. Existing project hook
entries and unrelated settings remain preserved.
